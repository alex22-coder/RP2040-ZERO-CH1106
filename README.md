# Подключение и работа с дисплеем 1.3" (SH1106) на RP2040

Данный проект демонстрирует подключение OLED-дисплея 1.3 дюйма с разрешением 128×64 на основе контроллера SH1106 к плате на базе RP2040 (например, **RP2040-Zero**) по интерфейсу I2C. В качестве библиотеки для работы с дисплеем используется **U8g2**, которая предоставляет богатый набор функций для вывода текста, графики и шрифтов.

---

## Схема подключения

| Пин дисплея | Пин RP2040 |
|-------------|------------|
| VCC         | 3.3V       |
| GND         | GND        |
| SDA         | GPIO 4     |
| SCL         | GPIO 5     |

**Примечание:** В коде пины заданы явно через `Wire.begin()` без дополнительной настройки, так как библиотека U8g2 использует стандартный объект Wire. При необходимости можно переопределить пины с помощью `Wire.setPins(SDA_PIN, SCL_PIN)` (в коде закомментировано).

---

## Установка библиотек

Для работы потребуются следующие библиотеки (устанавливаются через менеджер библиотек Arduino IDE или PlatformIO):

- **U8g2** – основная библиотека для управления дисплеем.
- **Wire** – встроенная библиотека для I2C (уже есть в Arduino).

---

## Код программы

Ниже представлен базовый шаблон для инициализации дисплея и вывода текста.

```cpp
#include <Arduino.h>
#include <Wire.h>
#include <U8g2lib.h>

#define SDA_PIN 4
#define SCL_PIN 5

// Объявляем объект дисплея (SH1106, 128x64, аппаратный I2C)
U8G2_SH1106_128X64_NONAME_F_HW_I2C u8g2(U8G2_R0, U8X8_PIN_NONE);

void setup() {
  Serial.begin(115200);
  
  // Инициализация I2C на заданных пинах (опционально)
  // Wire.setPins(SDA_PIN, SCL_PIN);
  Wire.begin();
  Wire.setClock(400000);  // Скорость I2C – 400 кГц

  u8g2.begin();           // Запуск дисплея

  // Очищаем внутренний буфер
  u8g2.clearBuffer();
  
  // Устанавливаем шрифт (например, 8pt)
  u8g2.setFont(u8g2_font_ncenB08_tr);
  
  // Рисуем строки текста
  u8g2.drawStr(0, 15, "Hello RP2040-Zero!");
  u8g2.drawStr(0, 35, "   I2C pins 4/5");
  u8g2.drawStr(0, 55, "Device Address 0x3C");
  
  // Отправляем буфер на дисплей
  u8g2.sendBuffer();
  
  Serial.println("Display initialized");
}

void loop() {
  // Здесь можно разместить код для обновления данных
}
```
Основные функции библиотеки U8g2
Библиотека U8g2 предоставляет множество методов для работы с дисплеем. Ниже приведены самые часто используемые.

Инициализация

cpp
`U8G2_SH1106_128X64_NONAME_F_HW_I2C u8g2(U8G2_R0, U8X8_PIN_NONE);`
U8G2_R0 – ориентация экрана (0°).

Второй параметр – пины (при аппаратном I2C не используются).

cpp
`u8g2.begin();`
Запускает дисплей и настраивает I2C.

Управление буфером
`u8g2.clearBuffer() – очищает внутренний буфер (все пиксели выключены).`

`u8g2.sendBuffer() – отправляет содержимое буфера на дисплей (отображение).`

Работа со шрифтами
`u8g2.setFont(const uint8_t *font)` – устанавливает шрифт для последующего вывода текста. Доступны множество шрифтов (например, u8g2_font_ncenB08_tr, u8g2_font_6x10_tf, u8g2_font_helvB10_tr и др.). Полный список можно найти в документации U8g2.

Вывод текста
`u8g2.drawStr(x, y, "текст")` – выводит строку текста начиная с координат (x, y). Координата y – это базовая линия шрифта (нижняя граница текста).

`u8g2.setCursor(x, y)` – устанавливает позицию для последующего вывода через print() (подобно Serial).

`u8g2.print(данные)` – выводит данные (числа, строки) в текущую позицию курсора.

Рисование примитивов
`u8g2.drawPixel(x, y)` – зажигает один пиксель.

`u8g2.drawLine(x1, y1, x2, y2)` – рисует линию.

`u8g2.drawRect(x, y, w, h)` – рисует прямоугольник (только контур).

`u8g2.drawBox(x, y, w, h)` – залитый прямоугольник.

`u8g2.drawCircle(x, y, radius)` – окружность.

`u8g2.drawDisc(x, y, radius)` – залитый круг.

Управление пикселями
`u8g2.setDrawColor(color)` – устанавливает цвет рисования (1 – включить пиксель, 0 – выключить).

Полный цикл обновления

Обычно используется паттерн:

cpp
```
u8g2.firstPage();
do {
  // Все команды рисования здесь
  u8g2.drawStr(0, 10, "Hello");
} while (u8g2.nextPage());
```
Этот метод позволяет экономить память при использовании буфера ограниченного размера, но для простых задач можно использовать `clearBuffer/sendBuffer`.

Расширение функциональности
Вывод переменных
cpp
```
int value = 42;
char buffer[16];
sprintf(buffer, "Value: %d", value);
u8g2.drawStr(0, 20, buffer);
```
Работа с несколькими шрифтами

cpp
```
u8g2.setFont(u8g2_font_6x10_tf);
u8g2.drawStr(0, 10, "Small");
u8g2.setFont(u8g2_font_helvB14_tr);
u8g2.drawStr(0, 30, "Large");
```

Примечания
Адрес I2C дисплея по умолчанию – `0x3C` (для SH1106 часто используется 0x3C или 0x3D). При необходимости можно изменить в настройках библиотеки.

Если дисплей не инициализируется, проверьте правильность подключения пинов и наличие подтягивающих резисторов на линиях SDA/SCL (обычно они уже есть на модуле).

Для увеличения скорости обновления можно повысить частоту I2C до 800 кГц (если дисплей поддерживает).
